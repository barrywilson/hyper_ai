# Agent Instructions

This project has a context framework in `memory/`. Read these files at the start of each conversation:

- `memory/ABOUT.md` — project purpose, personas, and constraints
- `memory/ARCHITECTURE.md` — system design, patterns, and key decisions
- `memory/IMPLEMENTATION.md` — tech stack, languages, dependencies, and preferences
- `memory/ENV_SCRIPTS.md` — environments, commands, and developer scripts
- `memory/WORKFLOW_STATUS.md` — the four-stage development process and current status

Load all five before asking any questions about the project. For implementation tasks, `WORKFLOW_STATUS.md` defines the process to follow — read it before planning any work.

When you discover something about the project that isn't captured in these files, update the relevant one.
